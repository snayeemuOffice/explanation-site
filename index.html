How It Works

1. A consultant starts a scoping session via the React panel → main.py creates a Recall.ai bot.
2. The bot joins the meeting and streams transcripts to POST /webhook/recall.
3. Each utterance flows through the orchestrator.py pipeline:
    • Filter noise (filler words, short utterances)
    • Retrieve relevant KB chunks via vertex_client.py (RAG)
    • Extract structured requirements via claude_client.py (Claude Sonnet)
    • Apply hallucination guardrails (KB grounding, module scope, confidence floor)
    • Persist to Firestore and broadcast to the React panel via Redis → SSE
4. When the call ends, the session is marked complete and the post-call review screen appears.
Three Agentic Fixes:
Fix 1 — Rolling Summary: Compresses older utterances every N utterances to keep the context window bounded.
Fix 2 — Semantic Chunking: Filters filler words before calling Claude, reducing API calls by ~30%.
Fix 3 — Hallucination Guardrails: Three independent checks catch Claude's false extractions.

Data Flow

Incoming: Recall.ai webhook → main.py → orchestrator.py → vertex_client.py + claude_client.py
Storage: session_store.py (Redis, ephemeral) + firestore_client.py (Firestore, permanent)
Outgoing: redis_broadcast.py (Redis pub/sub → SSE → React dashboard)
